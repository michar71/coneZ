Cues
====
- All Cues are 64 Bytes long
- Cues ar timestamped event-point that trigger actions/effects or set parameters.
- If thd timestamp is inthe past (or 0) the cue is executed immidiatly, otherwsie at the time specified.
- If the duration is 0 the cue will (Depending on the type) run indefentily/until overriden by another cue (Or self-terminates).
- Channel 0 is special. Any effect/parameter here will be applied to all Channels
- Parmeter ID 0 is special. It is used for Script control. 
  - Value 0 -> NOP (So parameter 0, Value 0 does nothing)
  - Value 1 -> Restart Script
  - Value 2 -> Pause Script
  - Value 3 -> Continue Script
  - Value 4 -> Abort Script

General:
Times are expressed as Unix Epoch Time (Long Long, 8 Byte, 64 bits)
Channels 0..65535 (2 bytes, 16 bits)
Parameter 0..65535v (2 Bytes, 16 Bits)
Type 0..65535 (2 Bytes, 16 Bits)

Data common to all Cues:
Timestamp
Duration
Type
Channel

(Total Heeader Size: 8+8+2+2 = 20 Bytes)

Cue-List
========
If a whole list of cues is transmitted its csalled a cue-list. (Duh...)
- Cue-lists can be stored as files (With the. suffix .clf for cue-list file)
- Cue lists chan be opened in chunks of 64 bytes and traversed
- A cue list shall have 1 header (64 bytes) at the beginning followed by 64-byte cues
- Cues in the cue list shall be sorted by timestamp
- A cue list header is just a special type of cue.....


Cue-List Header
---------------
(20 byte cue header, global time to activate cue list as start, media duration a length, Channel 0)
Cue List ID (32 Bit, 4 bytes) -> Unique Cue List ID
Cue-Count (32 Bit 4 Bytes) -> Number of cues in list or 0 if undetermined (Read filesize in that case)
Media Link (20 bytes). -> String 
Expiration Time (64 Bit, 8 Byte) -> Time when the list can be discarded
Version (2 Byte) -> Version of Cue List/Cue Protocol



Status Report from Device
-------------------------
Not really a cue but fits into 64 Bytes.

- Timestamp (8 Bytes)
- ID  (4 Bites)
- Lat (8 Bytes)
- Long (8 Bytes)
- Alt (2 Bytes)
- Status (2 Bytes)
- Activity (2 Bytes)
- Battery (2 Bytes)
- Radio Reception (2 Bytes)
