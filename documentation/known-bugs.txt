ROUND 3 FIXES (2026-04-19)
==========================

HIGH
----

#61 FIXED -- http.cpp page_catf() truncation corrupted HTML output.
    When vsnprintf overflowed the page_buf, n returned the full
    intended length so the guard `page_pos + n < sizeof(page_buf)`
    evaluated false and page_pos did not advance. The next
    page_catf call overwrote the truncated bytes, producing
    garbled HTML (visible on /dir and the partitions page). Fixed
    by clamping n to what was actually written and advancing
    page_pos unconditionally.

#62 FIXED -- cue.cpp elapsed_ms wrapped after ~49 days of playback.
    uint32_t (now_ms - music_start_ms) overflowed, causing cues
    to mis-match or stop firing during very long sessions.
    Lifted elapsed_ms and effective_start to 64-bit throughout
    the comparison.

#63 FIXED -- wasm_wrapper.cpp leaked host state on m3_RunStart failure.
    File slots, string pool, low heap, and wasm_current_path
    weren't reset on the m3_RunStart error path, so the next run
    inherited dirty state. Factored cleanup into
    wasm_cleanup_runtime() and called it from both the error and
    success paths.

#64 FIXED -- conez_wifi.cpp WiFi byte counters could tear on dual-core.
    volatile uint32_t s_tx_bytes/s_rx_bytes were load-add-stored
    from concurrent LWIP callbacks; a preempting core could lose
    an increment. Switched to std::atomic<uint32_t> with
    fetch_add(memory_order_relaxed).

MEDIUM
------

#65 FIXED -- c2wasm cw_malloc/realloc/calloc returned NULL on OOM.
    Many call sites used the result without checking. Wrapped both
    the embedded (cw_fatal + longjmp) and standalone (fprintf +
    exit) allocators to abort on NULL; callers are now guaranteed
    a valid pointer.

#66 FIXED -- c2wasm buf.c capacity doubling could overflow int.
    `while (nc < b->len + need) nc *= 2;` on a large-enough
    capacity wrapped to a negative value, causing an infinite
    loop or allocation failure. Added INT_MAX/2 guard that
    calls cw_fatal.

#67 FIXED -- WASM file_seek accepted any offset without validation.
    A negative or out-of-range offset could leave the FILE* in
    an undefined state. Now computes the target via ftell+fstat
    and rejects offsets that would move outside [0, filesize].

#68 FIXED -- psram.cpp had 8 unbounded `while (GPSPI2.cmd.X) ;` spins.
    If the SPI2 peripheral hung, the calling task looped forever.
    Added SPI2_WAIT(cond) macro (1M-iter cap ~= 4ms at 240MHz)
    and replaced all 8 spins.

#69 FIXED -- Telnet IAC subnegotiation could lock a client in state 3.
    A peer sending unterminated IAC SB payload would consume
    bytes indefinitely. Added iac_sb_bytes counter per client,
    aborted after 256 bytes.

#70 FIXED -- NMEA RMC/GGA overwrote lat/lon on bad-fix modes.
    RMC mode E/N/S or GGA quality 6-8 correctly left
    s_has_fix=false, but position was still copied to d->lat /
    d->lon. Consumers that forgot to check location_valid saw
    garbage coordinates (e.g. from dead-reckoning drift).
    Now only overwrite lat/lon when s_has_fix is true; the
    last known good position is preserved.

#71 FIXED -- datetime_to_epoch_ms only validated month.
    Garbage day/hour/minute/second in NMEA produced wildly
    wrong epochs. Now range-checks year (1970-2099), month
    (1-12), day (1-31), hour (0-23), minute (0-59), second
    (0-60 for leap); returns 0 on invalid input.

#72 FIXED -- config_save and config_set_field had no locking.
    A cross-task reader could observe a half-written string
    field (wifi_ssid, mqtt_broker, etc.). Added config_mutex
    created in config_init; take/give around set_field (per
    write) and save (snapshot loop).

#73 FIXED -- editor_save was not atomic.
    A crash or power loss mid-save left the file truncated or
    empty. Now writes to <path>.tmp, flushes+closes, then
    renames into place; unlinks the tmp on any error.

LOW
---

#74 FIXED -- led_setup() could leak CRGB arrays on re-entry.
    Safe today (called only from main setup()), but a future
    config-reload path would leak the four buffers. Now
    delete[] the old arrays before re-allocating; rmt_init()
    guarded by a static bool so RMT channels and encoder can't
    double-create.

#75 FIXED -- led_push_hw's grb_buf was shared without synchronization.
    Only one caller today, but led_show_now() is public.
    Extended led_mutex to cover the whole led_push_hw, serializing
    against led_set_channel writers and any hypothetical
    concurrent pusher. rmt_transmit reads grb_buf asynchronously
    until rmt_tx_wait_all_done returns, so the buffer must stay
    intact for the full duration.

#76 FIXED -- WASM string pool and low heap fragmented over time.
    pool_free/low_heap_free only reclaimed the top allocation;
    adjacent freed blocks weren't coalesced. Long-running modules
    gradually exhausted the pool even when total free space was
    ample. Now coalesce forward and backward on free, shrink
    bump when the coalesced block is at the top, and prune
    trailing dead records (marked size=0, skipped by first-fit
    since min alloc is 4 bytes).

#77 FIXED -- printManager ANSI stripper only handled CSI.
    `ESC ]` (OSC — hyperlinks, window titles) and single-char
    ESC sequences (`ESC 7`, `ESC c`, etc.) landed in the log
    ring buffer / file sink / MQTT feed mangled. State machine
    rewritten to handle ESC [ ... final (CSI), ESC ] ... BEL|ST
    (OSC), and ESC X (single-char intermediate).

#78 FIXED -- effects.cpp function-scope statics persisted across switches.
    Transitioning between effects (or calling one after a long
    gap) left stale prev_sec / state / target_ms / step / offset_ms
    values that could trigger a spurious fire or skip. Moved all
    statics to module scope with s_* prefixes and added
    effects_reset_all() (declared in effects.h) to zero them.

ROUND 4 FIXES (2026-07-19)
==========================
Multi-agent code review of the full firmware tree. Fixes span the BASIC
interpreter, LED driver, WASM host layer, LoRa firmware-distribution
receiver, console/log, and config. Matching fixes were mirrored into the
Qt simulator (sim_wasm_imports_*, led_state.cpp) where the same code lives.

CRITICAL
--------

#80 FIXED -- BASIC array access gave scripts arbitrary heap read/write.
    bound() raised BOUNDS but still returned mem+n, and STOREI_/LOADI_
    dereferenced it before the driver checked globalerror. bound() now
    returns NULL on an out-of-range/non-array index and the opcodes bail.
    A(i) on an un-DIMed variable also compiled to STOREI_ with no mode
    check (NULL/scalar deref); opcodes now require a live DIM allocation.
    (basic.h)

#81 FIXED -- BASIC DIM double-free / free-of-scalar, within and across runs.
    freedim() freed on compile-time VARMODE_DIM, so a DIM in a dead branch
    -- or a re-DIM in the next program over the persistent symbol table --
    freed a stale or scalar value. Added runtime dim_alloc[] tracking:
    freedim() frees only real allocations and clears them, and initbasic()
    now frees leftovers and resets the symbol table per program.
    SETARRAY/SHIFTARRAY/ROTATEARRAY/SCALELIMITARRAY now validate with
    is_dim_ptr() instead of a bare !=0, closing an arbitrary-pointer
    read/write. SUB params/LOCALs bounded to LOCS; interp() no longer leaks
    the script FILE* on a compile error. (basic.h, basic_extensions.h)

#82 FIXED -- LED channel resize was a use-after-free.
    led_resize_channel() delete[]'d the old buffer outside led_mutex while
    ArtNet/BASIC/WASM/CLI writers held the raw leds* pointer with no lock.
    Buffers are now allocated once at boot to a fixed capacity; resize only
    moves the logical count within that capacity and never frees. Growth
    beyond the boot size returns -2 ("reboot to grow"). (led.cpp, commands.cpp)

#83 FIXED -- LoRa dist heap over-read from a crafted DIST_DATA chunk.
    finalize_block() read blk_comp_len / ulen bytes out of a blk_N*DIST_L
    buffer with no bound, so a chunk declaring a huge length forced an
    over-read (up to 64 KB from a 200-byte block). Now rejected. (dist.cpp)

#84 FIXED -- WASM printf/snprintf host imports leaked host stack / smashed a
    stack buffer. The copy loop ran to snprintf's would-be length (not the
    truncated count), so "%600d" read past tmp[] into host memory handed back
    to the guest; a long flag / *-width / precision run also drove the spec[]
    index past its buffer. Clamped the length and bounded spec[] (and added
    the -999..999 width/precision clamps). (wasm_format.cpp)

#85 FIXED -- WASM analog_read could brick PSRAM/LoRa SPI.
    adc_read_*/adc_ensure_channel accepted GPIO 1-10 and lazily configured
    any of them as analog; GPIO 4-7 (PSRAM) and 8-10 (LoRa) then lose SPI
    until power cycle. Restricted to GPIO 1-3 at the source (protects the
    WASM and BASIC analog_read and the CLI). (adc.cpp)

#86 FIXED -- WASM /config.ini write guard was bypassable.
    The exact-string check missed //config.ini and /./config.ini. Now rejects
    any path whose final component is config.ini. (wasm_imports_file.cpp,
    wasm_imports_compression.cpp, wasm_imports_deflate.cpp)

HIGH
----

#87 FIXED -- "load" held the print mutex across an unbounded, no-yield receive
    loop, stalling every printing task and starving the idle task into a WDT
    reset if a transfer hung. Now takes the lock only for the flush, polls
    with a yield + 30 s idle timeout, and writes a final line that has no
    trailing newline (previously dropped). (commands.cpp)

#88 FIXED -- "top" heap overflow when the task count grew mid-session.
    Only curr/order were reallocated on growth; the end-of-frame swap then
    made the smaller stale buffer the next curr and uxTaskGetSystemState()
    overran it. Now grows all three via realloc. (commands.cpp)

#89 FIXED -- log_open/log_close/log_free raced the vprintfnl sinks.
    Teardown ran without print_mutex, so a concurrent sink could fprintf/
    fclose a closing FILE*, or write to a freed ring and divide by a zeroed
    slot count. Lifecycle functions now take print_mutex. (printManager.cpp)

#90 FIXED -- MQTT publisher task could publish on a destroyed client.
    mqtt_force_disconnect() esp_mqtt_client_destroy()'d s_client while the
    publisher task held and used it. Added a client mutex serializing the
    publish against the destroy. (conez_mqtt.cpp)

#91 FIXED -- inflate underflowed on truncated gzip input.
    data_len = in_len - off - 8 with no check that >= 8 bytes remain, so a
    truncated file produced a huge size_t and a massive over-read. Guarded.
    (inflate.cpp)

#92 FIXED -- LoRa dist manifest parsed with strtok_r over a non-terminated
    buffer. rx_buf was malloc(flen); strtok_r scanned past the end on every
    manifest reception. Allocate flen+1 and NUL-terminate before parsing.
    (dist.cpp)

#94 FIXED -- "dir *.bas" (root wildcard) matched nothing. The filter pointer

#95 FIXED -- Load average pegged at 2.00 forever once the task count exceeded

#96 FIXED -- WASM file_readln one-byte stack overflow (line[256]='\0' when cap

#97 FIXED -- HTTP OTA reported success even if esp_ota_set_boot_partition

OPEN (reported this round, not yet fixed)
-----------------------------------------

#101 BASIC PRINTS and FORMAT "$" dereference an arbitrary stack value as a C
    string (info leak / crash from a one-character script mistake). A safe fix
    needs to validate the value points into the string table; the code path
    treats string values inconsistently (stab+offset for the format string vs
    a raw pointer for $ args), so it was deferred pending a closer look.

#102 FIXED -- GPS/time cluster (gps.cpp, nmea.cpp). Four issues:
    (a) the epoch anchor gated only on pps_count>0, so a module that stopped PPS
        but kept emitting NMEA time from its RTC made get_epoch_ms() run ahead
        without bound -- now also requires the PPS edge to be fresh (age <1.2s);
    (b) datetime_to_epoch_ms()==0 was stored unchecked (clock jumped to 1970) --
        now the anchor skips a zero result;
    (c) the anchor fired on any position update, so a GGA's new-day time paired
        with the previous RMC's date at UTC midnight and stepped back 24 h --
        nmea.cpp now bumps a datetime_count only on an RMC that carries a
        consistent date+time pair, and the anchor keys on that (never a GGA);
    (d) both ntp_loop() variants overwrote the epoch from the system clock even
        when SNTP had not synced this session -- since the ESP32 RTC survives
        soft resets, a stale >2024 clock clobbered the beacon/GPS/compile epoch.
        Now gated on ntp_last_sync!=0; unsynced, the display fields advance from
        the free-running epoch without overwriting it.

#103 FIXED (partly a mis-diagnosis) -- IMU tilt handling (sensors.cpp,
    mpu6500.cpp). CORRECTION: the original report claimed getRoll()'s +180
    offset was wrong and caused a permanent tilt alarm on every cone. That was
    based on a wrong assumption -- the PCB is mounted UPSIDE DOWN, so upright
    reads az = -1 g and the +180 is CORRECT (upright -> 0 deg). getRoll/getPitch
    were left as-is. The real, orientation-independent fixes: (a) the boot
    sanity check "in space" test now uses fabs(v_accZ) (works either way up);
    (b) mpu6500_calibrate() divides by the number of SUCCESSFUL samples (not a
    fixed 50) and targets +/-1 g on Z by the sign it actually reads, so running
    MPU_Calibrate on the upside-down device no longer flips the axis and creates
    the alarm; (c) mpu6500_read() returns success and sensors_loop() logs a
    sustained I2C failure instead of silently holding stale data (IMU is left
    uncalibrated in the field, so the calibration path was latent).

#104 FIXED -- Cue engine ordering + fix-validity (cue.cpp; mirrored in the
    simulator's cue_engine.cpp). The loop advanced a single monotonic cursor and
    broke at the first not-yet-due cue, but spatial offsets make effective start
    times non-monotonic, so one offset cue stalled every later cue. Replaced the
    cursor with a per-cue "fired" flag and a precomputed effective-start (offset
    is fixed for the run), scanning all cues each tick. cue_start() also gates
    spatial timing on a real GPS fix (get_gpsstatus()); with no fix it falls
    back to base start times instead of computing continent-scale offsets that
    would leave the cone dark.

#105 FIXED -- [HIGH] LUT double-free and TOCTOU heap overflow (util/lut.cpp,
    util/lut.h). lut.h exported pLUT/lutSize/currentLUTIndex as raw globals
    while lut_mutex was private, so BASIC's unlocked free(pLUT) raced
    loadLut()'s locked free and WASM's pLUT[index]=value -- double-free /
    use-after-free across BasicTask and WasmTask. The globals are now static;
    all access goes through locked accessors (lut_read/lut_write/lut_get_size/
    lut_current_index/lut_replace_from_array). loadLut()'s read loop is bounded
    by i<size against a file that changed since checkLut()'s count.

#106 FIXED -- [HIGH] Editor destroyed files longer than 512 lines on save
    (console/editor.cpp). editor_load() now sets a load_truncated flag when the
    file exceeds ED_MAX_LINES (or a PSRAM OOM cuts the load short), shows a
    READ-ONLY status, and editor_save() refuses so the original isn't
    overwritten with the truncated prefix.

#107 FIXED -- [HIGH] Unbounded dir/grep recursion on the 8 KB ShellTask stack
    (console/commands.cpp). dir_list/dir_max_name/grep_dir carried ~550 B of
    path buffers per frame and recursed to filesystem depth. Added a
    MAX_DIR_DEPTH (12) cap to all three.

#108 FIXED -- deflate LZ match finder was inert (util/deflate.c). Three coupled
    issues: (a) the current position was inserted as the hash-chain head before
    the search, and that head is also find_match's "candidate != dict_pos" loop
    sentinel, so the loop never ran, find_match always returned 0, and every
    byte was emitted as a literal (valid gzip but Huffman-only, ~2:1). Now
    find_match skips the just-inserted head (starts from NEXT). (b) Once matches
    fire, overlapping matches (dist < len) compared the lookahead against dict
    bytes at/after the write pointer that are stale -- now compared against the
    lookahead itself (cur_buf[len-dist]). (c) build_tree's max-bits
    oversubscription check computed a floored Kraft (repeated >>1) and missed
    every 1 < Kraft < 2 case; worse, a naive "reduce below 2^max_bits" fix left
    the lit/len tree INCOMPLETE (decoder rejects it). Replaced with the exact
    zlib gen_bitlen redistribution that brings the code to exactly complete.
    Verified with a host round-trip harness (PSRAM stub + gzip_buf) over
    empty/tiny/overlap/text/random/500KB/Fibonacci-skew/self-source inputs at
    all levels: every stream decompresses via system gunzip, and compressible
    data now shrinks ~10-300x (e.g. 500 KB repeated text 287 KB -> 1.3 KB,
    a real source file 62% -> 29%) where it was ~58-62% before.

#109 Console/shell robustness (console/): telnet client slots are mutated from
    the unlocked line_editor read path (can send to a reused fd); usb_write
    blocks 10 ms under print_mutex when no USB host is attached; the file-log
    sink does flash fprintf+fflush while holding print_mutex; the shell/editor
    CSI parsers leak VT220/xterm function-key bytes into the line/file; Ctrl+D
    farewell sends 5 of the 6-byte reset; telnet IAC/SE is lost across a recv
    boundary; log_save/log_show read the ring without print_mutex (torn
    entries).

#110 CLI argument handling (console/commands.cpp): cmd_lora writes user values
    into config before the driver validates them (bad RF settings persist on
    the next save); dir silently caps a directory at 32 entries; cmd_deflate
    treats a digit-leading output filename as the compression level; grep
    truncates the pattern to 63 chars; cmd_game puts ~1.8 KB of grid arrays on
    the ShellTask stack.

#111 PSRAM raw transfers cross the LY68L6400 1 KB page boundary
    (psram/psram.cpp). Chunks start at arbitrary offsets and are never split at
    1 KB, so an odd 512-byte cache page straddles a boundary; if the chip wraps
    the burst within the page the tail bytes hit wrong addresses -- silent
    corruption the 64-byte-aligned self-test can't detect. CE# low time also
    exceeds the chip's tCEM budget at full chunk size, and psram_free() on an
    untracked heap-fallback address can double-free.

#112 Sensor/alarm quality (sensors/): GSA fix-type/PDOP/VDOP are committed
    before checksum validation; ADC error paths return 0 mV indistinguishable
    from a real reading (feeds the low-battery alarm, no hysteresis);
    mpu6500_calibrate() divides by a fixed N despite skipping failed reads; fix
    loss with empty position fields relies only on the 10 s staleness timeout;
    get_epoch_ms() elapsed math wraps if the anchor is >49.7 days old.

#113 Config locking / web form (config.cpp, util/pm.cpp): config_save opens/
    truncates /config.ini before taking the lock and has no temp+rename;
    several struct writes skip CONFIG_LOCK; the web form can't clear a string
    setting (empty values are discarded); pm.cpp s_lock_count is non-atomic.
